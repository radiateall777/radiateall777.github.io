Git介绍

Git是一个免费的、开源的分布式版本控制系统，可以快速的处理从小型到大型的各种项目。

何为版本控制

版本控制是一种用于记录文件内容变化，以便将来查阅特定版本修订情况的系统。

版本控制最重要的是可以记录文件修改历史记录，从而使用户可以看到历史版本方便版本切换。

为什么需要版本控制

个人开发过渡到团队协作（多副本的方式是有很大缺陷的，很容易导致两个人的代码相互覆盖）

版本控制工具
  1. 集中式的版本控制工具

像CVS、SVN（Subversion）、VSS······这一类的软件属于集中式的版本控制工具。这些都有一个单一的集中管理的服务器，保存所有文件的修订版本，协同工作的人们通过客户端连接到这台服务器，取出最新的文件或者提交更新。

  1. 分布式版本控制工具

像git这一类的分布式版本控制工具，客户段提取的不是最新版本的文件快照，而是把代码仓库完整的镜像下来(本地库)。这样任何一处协同工作用的文件发生故障时，事后都可以用对其他客户端的本地仓库进行恢复，因为每个客户端的每次文件提取操作，实际上都是对整个文件仓库的完整备份。

优点：

  • 服务器断网的情况下也可以进行开发（因为版本控制是在本地进行的）
  • 每个客户端保存的也都是整个完整的项目（包含历史记录，这样更加安全）

Git和代码托管中心

代码托管中心是基于网络服务器的远程代码仓库，一般我们称为远程库。

  • 局域网
    • Gitlab
  • 互联网
    • GitHub（外网）
    • Gitee码云（国内网站）

Git常用命令

设置用户签名

git config –global user.name 用户名。 设置用户签名

git config –global user.email 邮箱。 设置用户签名

注意：

  • 签名的作用是区分不同的操作者身份。用户的签名信息在每个版本的提交信息中都能查看得到，以此确认本次提交是谁做的。Git首次安装必须设置一下用户签名，否则无法提交代码。
  • 这里设置的用户签名与将来登陆github的账号没有任何关系。
版本号的穿梭

git reset –hard 要穿梭的版本号

Git 分支操作

  1. 什么是分支

​ 在版本控制过程中，同时推送多个任务，为每个任务，我们就可以创建每个任务单独的分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候不会影响主线分支的运行。分支可以简单的理解为一个副本，一个分支就是一个单独的副本。（分支底层实际上也是指针的引用）

  1. 分支的操作

    • 查看分支
    • 创建分支
    • 切换分支
    • 把指定的分支合并到当前开发的分支上

注意：

冲突合并：合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。Git无法帮我们决定使用哪个。必须人为决定新代码的内容。

GitHub操作

  1. 创建远程仓库 —— New Repository

​ 注意：远程库和本地库的名字最好保持一致

  1. 远程仓库操作

+ 拉取远程库到本地库

​ 基本语法：git pull 远程仓库别名 远程分支名

​ 一定要保证本地库的代码与远程库是同步状态，要及时更新自己的本地库代码。

+ 克隆远程仓库到本地库

​ 基本语法： git clone 远程地址

​ 三件事：1）拉取代码 2）初始化本地仓库 3）创建别名

+ Git_GitHub_团队内协作

​ 在需要团队协作的库中，点击collaborators下的Mannge access邀请团队协作的成员

+ Git_GitHub_跨团队协作

​ 首先需要先进行fork，叉一份到你的仓库，然后克隆到本地库进行修改，或者直接进行修改，修改好以后，需要点击pull request创建一个pull request请求，发送给开发者。 

+ Git_GitHub_ssh免密登陆

​ 远程仓库一般有一个SSH地址，因此我们可以使用SSH进行访问

​ 首先运行命令生成.ssh密钥目录：使用 ssh-keygen -t rsa -C 账号邮箱（rsa是一种非对称形式的加密协议）

​ **Mac上使用shirt + command + .**显示隐藏的文件

​ 打开生成的.ssh目下的pub文件会看到一个公共密钥

Idea集成Git

  1. 配置一个Git的忽略文件

​ 原因：所忽略的文件与项目的实际功能无关，不参与服务器上部署运行。把他们忽略掉可以屏蔽IDE工具之间的 差异。

​ 方法：创建忽略规则文件xxxx.ignore(前缀名可以随便起，建议是git.ignore)

​ 这个文件原则上放在什么地方都可以，但为了便于～/.ignore文件引用，建议也放在用户家目录下。

Idea中集成GitHub

使用token进行登陆，在GitHub中点击settings在开发人员设置中的personal access tokens中生成一个登陆口令，权限全部给满。

注意，push是将本地代码推送到远程库，如果本地代码与远程库代码不一致，push的操作会被拒绝，也就是说，要想push成功，一定要保证本地库的版本要比远程库的高。因此在动手改代码之前，一定要先检查本地库代码与远程库代码的区别，如果本地代码版本落后，就需要首先pull拉取远程库的代码，将本地代码更新到最新后，然后再修改提交推送。
